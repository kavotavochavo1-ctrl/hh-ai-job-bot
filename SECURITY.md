# Security

## Секреты и приватные данные

Никогда не добавляйте в Git:

- `.env`;
- `hh_storage_state.json` и другие файлы браузерной сессии HH;
- `bot.db` и другие SQLite-файлы;
- токены Telegram-бота;
- ключи OpenRouter;
- реальные данные работодателей и переписок.

Если ключ или токен случайно опубликован, немедленно отзовите его у
соответствующего провайдера и выпустите новый. Удаления строки из следующего
commit недостаточно: секрет останется в истории Git.

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости в открытом issue. Свяжитесь с владельцем
репозитория через контакты профиля GitHub и дайте время на проверку и
исправление.
